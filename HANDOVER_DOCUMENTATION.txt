# PORTFOLIO PROJECT HANDOVER & DOCUMENTATION

This document provides a technical overview of the components and design patterns used in this portfolio website.

---

## 🚀 Core Technologies
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS (with CSS Variables/Design Tokens)
- **Animations**: Framer Motion
- **State Management**: React Hooks & Context API

---

## 🎨 Design System & Theming
- **Location**: `app/globals.css` and `app/context/ThemeContext.tsx`
- **Why**: Centralized design tokens (colors, gradients, shadows) ensure consistency. 
- **How**: We use CSS variables like `--color-primary` which are mapped to Tailwind classes like `bg-primary`.
- **ThemeContext**: Provides `isLight` state to the whole app, allowing components to toggle between light and dark aesthetics dynamically.

## 📦 Component Breakdown

### 1. Hero (`app/components/Hero.tsx`)
- **What**: The landing section of the page.
- **Why**: Creates the first impression with a dynamic, interactive 3D orbit and personalized typing effect.
- **How**: 
  - **Typing Effect**: Uses `useEffect` and `useState` with `setInterval` to cycle through strings from `constants/typingTexts.ts`.
  - **3D Tech Orbit**: Uses Framer Motion's `useAnimationFrame` to rotate icons in a circular path. Each icon is a `motion.div` with custom `z` index and `scale` based on its "depth" in the rotation.
  - **Centering**: Icons use `translateX: -50%, translateY: -50%` to ensure they are perfectly centered on their orbit path.

### 2. About Me (`app/components/About.tsx`)
- **What**: Professional introduction and developer statistics.
- **Why**: Provides a high-density information layout (Stats, Focus Areas, Skills) to establish credibility.
- **How**: 
  - **Grid Layout**: Responsive grid that stacks on mobile and expands to a 3-column layout on large screens.
  - **Entrance Animations**: Uses `staggerChildren` from Framer Motion to animate items in sequence as they scroll into view.

### 3. Tech Stack / Skills (`app/components/Skills.tsx`)
- **What**: A standalone section focused on the technology icons.
- **Why**: Visually showcases the breadth of technologies used in a more immersive way than a simple list.
- **How**: 
  - **Multi-Ring Orbit**: Similar to the Hero section but with multiple radii (Inner, Middle, Outer) rotating at different speeds.
  - **Mobile Optimization**: On mobile, the "TECH STACK" text is hidden to save vertical space, and the orbit is specifically centered with fixed container dimensions.

### 4. Projects (`app/components/Projects.tsx`)
- **What**: A parallax gallery of work.
- **Why**: Highlights key projects with smooth, high-end transitions that feel professional.
- **How**: 
  - **Scroll Links**: Uses `useScroll` from Framer Motion to tie component animations directly to the scroll position.
  - **Vertical Stack**: The main container is tall (`400vh`), and children cards use `sticky` positioning to "stay" on screen while they animate their properties (scale, opacity) based on the scroll progress.

### 5. Learning Journey / Timeline (`app/components/Timeline.tsx`)
- **What**: Chronological view of development experience.
- **Why**: Shows growth and commitment to continuous learning.
- **How**: Leverages `useInView` to trigger slide-in animations when the user reaches the section.

### 6. Contact & Footer
- **What**: Social links and CTA.
- **Why**: Provides clear routes for communication.
- **How**: Custom hover states use branded hex colors (e.g., GitHub black, LinkedIn blue) in a transition to give the UI a polished feel.

## 🛠️ Usage Patterns for Future Development

### Adding a New Skill Icon
1. Add the SVG logo to `public/logos/`.
2. Update the `technologies` array in `app/components/Skills.tsx` or wherever the icons are mapped.

### Updating Design Tokens
1. Modify the CSS variables in `app/globals.css`.
2. Ensure you update both `.light-theme` and the root (dark) styles to maintain theme compatibility.

### Centering Elements
- Always prefer `flex items-center justify-center` for standard layouts.
- For absolute positioned elements (like orbits), use `top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2` for pixel-perfect anchoring.
